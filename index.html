My Cheatsheet

Learning to build websites can be frustrating, especially when you want it to look and function a certain way but can't recall the correct syntax. That is why writing your own cheatsheet makes perfect sense!

So below are a few examples of syntax rules for HTML and CSS that are always useful to remember:

HTML Semantic and Unsemantic Tags

Semantic tags are a naming convention for HTML elements where the writer uses a descriptive identifier for the content inside the element, without being too specific or too vague. This naming convention improves readability for screen reader programs or other developers and prevents a lot of wasted time changing multiple element names in the event of a future redesign.

Semantic

  • <article> tag

    Using the <article> tag in place of a <div> tag for the main informational part of a webpage shows developers what the section will contain without specifying any information.

  • "success" class

    labelling a message as success or similar allows the developer to see that the element should contain a positive confirmation message which displays to the user. An improvement on this would be adding a minor descriptive term, e.g "successText" to fully convey the intent without specifying any information.

  • "priority_1" class

    This may seem too specific, however many companies like to specify an elements importance in realtion to other elements. (e.g button importance). So while it looks unsemantic, it is a common convention.

Unsemantic

  • "normal" class

    The use of any name like "normal" is too open to interpretation, and therefore make no real description of the content inside an element. it is better to use the standard HTML tag names for elements you consider "normal".

  • "left" class

    This name is far too specific and in any future redesign, if you change the content to "right" (for example left aligned to right aligned) all uses of "left" would have to be changed. Best to use "alignment" for any align related elements to reduce potential edit time.

  • "introp" id

    As explained in an example found on a CSS/HTML semantics website, use of "introp" clearly means "introduction paragraph", however it is not a good idea to include the element being named.

HTML Inline Elements

Inline elements are those applied directly inside another element, applying an additional effect. Some examples of these are:

  • <span>

    An "inline container" used to group elements.

    <span style="color: blue">Style this text</span>.

    The font inside the element where span has been used will be styled to blue.

  • <mark>

    Used to highlight a section of text

    <mark>Highlight this text</mark>

    The text inside the element will be highlighted yellow (as default).

  • <sub>, <sup>

    The sub tag will convert the contents to a subscript while the sup tag will convert to a superscript.

    x<sub>2</sub> AND x<sup>2</sup>

    The text using sub will read x2 while the text using sup will read x2.

  • <address>

    Used to separate address information from the surrounding content. while it doesn't appear to have a different function to styling the address in italics, it can be quickly identified by screenreaders and other programmers by using this tag. largely a semantic use.

    <address> 7 Spinsters Box, Drychute Road, Lonelee </address>

    The address will be on a separate line

  • <code>

    The code tag will change the text to "code format". the box below is using the code tag (with additional CSS styling).

    <code>example of use</code>

CSS Combinators

"Combinators" are the syntax used to write multiple selectors for styling multiple elements, and the combinator needed is dependent on the hierarchy of the elements in the HTML.

Name of Combinator Explanation Syntax
Adjacent Sibling Specifies that both elements selected have the same parent and that B immediately follows A A + B { }
General Sibling Specifies that both elements selected share the same parent, and that A comes before B somewhere. A ~ B { }
Child Specifies element B is the direct child of element A. A > B { }
Descendant Specifies that element B is a descendant of A but not necessarily a direct child. A B { }
Column Specifies that element B is located within the table column specified by A. elements spanning multiple columns are considered members of ALL these columns. A || B { }
